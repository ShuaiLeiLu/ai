# Cache All Page Sections

## Objective

把所有页面版块的数据纳入缓存策略，页面读接口直接读取缓存数据；外部行情、资讯、LLM 或昂贵聚合由后台刷新或写操作失效后再生成。

## Scope

- 页面前端实际调用的后端读接口。
- 外部数据版块：盘前、资讯分析、题材掘金、交易行情。
- 数据库版块：研究员工作台/市场、社区、研报、任务、生态、计费。
- 写操作必须失效相关缓存，避免页面长期读取旧数据。

## Validation

- 针对每个新增缓存读路径，有测试证明缓存命中时不执行原始查询或实时请求。
- 后端测试通过，已知缺失的 `app.modules.trading.rqalpha_adapter` 需单独记录。
- 最终审计前端 API 调用列表和后端路由，确认页面读路径均有缓存策略。
