# 赛博投研产品体验审计

## 1. 文档信息

- 文档名称：赛博投研产品体验审计
- 审计日期：2026-04-17
- 审计对象：[赛博投研](https://mcp.jinhongwanli.com/)
- 审计方式：基于登录后的真实页面访问、前端路由与前端打包代码分析、部分接口验证
- 账号类型：普通注册用户（非 VIP）

## 2. 审计目标

- 标记当前产品中哪些模块为真实可用功能
- 标记哪些模块为占位、演示、前端本地样例或开发中
- 评估真实功能的完整度、可操作度与风险提示质量
- 为后续产品迭代和 PRD 优先级修正提供依据

## 3. 判定标准

### 3.1 真功能

满足以下大部分条件：

- 页面可真实访问
- 有真实后端数据或真实接口
- 可完成核心操作链路
- 非纯前端静态示意
- 不只是“开发中”提示页

### 3.2 占位 / 演示

满足以下任一特征：

- 页面仅展示“开发中”“敬请期待”
- 页面数据为写死的前端本地假数据
- 页面仅做 UI 演示，无后端闭环
- 核心动作不可执行或无真实落库/流转

## 4. 总体结论

### 4.1 结论摘要

赛博投研当前不是一个纯演示站，而是“主链路已具备真实功能、部分外围模块仍处于占位或半成品状态”的产品。

当前最完整的真实主链路是：

- AI研究员工作台
- 人才市场
- 研究员编辑器
- 文档详情
- 盘前速览
- 资讯分析
- 赛博社区
- 我的文件夹 / 笔记 / 音频转写链路
- Webhook 配置页
- 会员、电池、邀请等账户体系

当前明显仍为占位/演示/未完成的部分是：

- 因子洞察
- 任务列表页面展示实现
- 回测复盘的大部分能力
- 部分盘前速览细分卡片可能仍混有示意数据

## 5. 模块审计明细

### 5.1 AI研究员工作台

- 判定：`真功能`
- 依据：可登录访问；可展示已雇佣研究员；可展示热门文档；存在真实文档详情与研究员相关接口
- 完整度：`高`
- 说明：这是当前产品最核心、最成熟的页面之一

### 5.2 文档详情

- 判定：`真功能`
- 依据：可通过真实文档 ID 访问；展示标题、任务进度、正文、结构化分析、风险提示等内容
- 完整度：`高`
- 说明：真实承载研究报告消费，是平台的重要价值点

### 5.3 人才市场

- 判定：`真功能`
- 依据：存在研究员列表、雇佣、解雇、模板查看、简历查看逻辑；前端代码与接口均真实存在
- 完整度：`高`
- 说明：已形成基础市场闭环

### 5.4 研究员编辑器

- 判定：`真功能`
- 依据：支持提示词设定、技能/知识库/MCP 选择、自驱任务、测试对话、保存
- 完整度：`高`
- 说明：属于生产型配置台，不是单纯页面示意

### 5.5 盘前速览

- 判定：`大体为真功能，局部疑似混合示意数据`
- 依据：页面结构完整，热讯、异常、趋势等展示真实；部分源码中存在示意数据痕迹
- 完整度：`中高`
- 说明：用户感知是真功能，但建议后续核查每个卡片是否都已切为真实数据源

### 5.6 资讯分析

- 判定：`真功能`
- 依据：存在资讯列表、分类筛选、热门股票联动、AI 解读入口与相关接口
- 完整度：`高`
- 说明：是与盘前速览并列的高频内容入口

### 5.7 模拟交易

- 判定：`真功能（内测态）`
- 依据：页面接入真实接口和 SSE；可展示交易组、账户摘要、持仓、日志、历史记录
- 完整度：`中高`
- 说明：功能真实，但产品定位仍明确为内测和观察用途

### 5.8 赛博社区

- 判定：`真功能`
- 依据：存在帖子列表、搜索、发帖弹窗、详情页、评论列表与相关接口
- 完整度：`高`
- 说明：内容生态已初步成型

### 5.9 我的文件夹 / 我的笔记

- 判定：`真功能`
- 依据：存在笔记、文件夹、上传 Token、音频上传、转写状态、AI 分析等真实逻辑与接口
- 完整度：`高`
- 说明：这是平台最容易被低估但实际非常关键的长期沉淀模块

### 5.10 Webhook

- 判定：`真功能（当前账号受 VIP 限制）`
- 依据：存在真实配置接口、研究员可选接口、权限判断接口、保存接口
- 完整度：`中高`
- 说明：普通用户无法完整体验，但不是占位页

### 5.11 我的知识库

- 判定：`基础真功能，深度能力待补`
- 依据：可见真实入口与创建知识库能力框架；与研究员编辑器绑定逻辑存在
- 完整度：`中`
- 说明：基础骨架是真，但深度内容管理能力还有扩展空间

### 5.12 技能市场 / 我的技能

- 判定：`真功能（早期阶段）`
- 依据：存在技能市场列表、搜索、我的技能上传入口、与研究员编辑器挂载关系
- 完整度：`中`
- 说明：已有真实供给结构，但内容规模较小

### 5.13 MCP市场 / 我的MCP

- 判定：`真功能（早期阶段）`
- 依据：存在市场列表、分类筛选、我的 MCP 管理入口、与研究员编辑器绑定关系
- 完整度：`中`
- 说明：生态框架已真实存在，但供给数量还少

### 5.14 回测复盘

- 判定：`半成品 / 低完成度真功能`
- 依据：有“我的策略”“新建”等页面结构，但能力闭环较弱
- 完整度：`低`
- 说明：更像预留中的后续模块，不适合当前作为主要卖点

### 5.15 因子洞察

- 判定：`占位 / 开发中`
- 依据：页面直接标识“功能开发中”
- 完整度：`极低`
- 说明：典型占位页

### 5.16 任务列表

- 判定：`演示 / 前端本地假数据`
- 依据：前端实现中任务列表数据写死在本地 store，非真实后端任务数据
- 完整度：`低`
- 说明：视觉上像真功能，但严格来说目前更偏演示页

### 5.17 会员套餐 / 电池明细 / 邀请体系 / 个人中心

- 判定：`真功能`
- 依据：存在真实页面、真实字段、真实电池流水与邀请规则展示
- 完整度：`中高`
- 说明：商业化基础能力已相对完整

## 7. 体验层面的主要问题

### 7.1 真功能与演示能力混杂

- 用户从页面上不容易区分哪些是生产功能，哪些是占位或演示
- 这会削弱产品可信度

### 7.2 “任务编排”价值被稀释

- PRD 中任务编排是关键能力
- 但现网“任务列表”页面当前偏演示，容易与真实自驱任务能力形成认知冲突

### 7.3 赛博实验室信息架构稍显分散

- 创建研究员、人才市场、技能市场、MCP市场、我的知识库之间关系对新用户不够直观

### 7.4 高级能力有真实基础，但缺少统一叙事

- 技能、知识库、MCP、Webhook 都是真能力
- 但在产品叙事上还没有被清晰包装为“研究员增强系统”

## 8. 审计建议

### 8.1 对外表达建议

优先对外宣传：

- AI研究员
- 研究文档
- 盘前速览
- 资讯分析
- 模拟交易（强调内测与观察）
- 我的文件夹

谨慎宣传：

- 任务列表
- 回测复盘
- 因子洞察

### 8.2 产品层建议

- 对开发中能力统一打上“开发中”或“内测”标识
- 对演示页与真功能页做明确视觉区分
- 将研究员增强能力统一包装为“技能 / 知识 / 工具”三位一体系统
- 重新梳理任务编排入口，避免真能力与演示页混淆

### 8.3 研发层建议

- 优先把任务列表切换到真实后端任务数据
- 补齐回测复盘能力的最小闭环
- 盘前速览各卡片统一数据口径与真实来源
- 为技能市场、MCP市场增加更明确的消费闭环指标

## 9. 最终结论

赛博投研当前已经具备较强的真实产品基础，不属于“只有 UI 的演示项目”。其最强部分是 AI研究员主链路与内容沉淀链路，最弱部分是部分实验性模块与任务展示能力。

一句话总结：

- `主链路是真功能，并且已经较完整`
- `外围生态是真框架，但还在早期`
- `个别模块仍是占位或演示，需要在对外叙事和内部排期中明确区分`
