# 模块 PRD：AI研究员工作台

## 1. 模块定位

AI研究员工作台是赛博投研的核心入口模块，负责承接用户进入产品后的第一层价值感知。它聚合已雇佣研究员、热门研究文档、公开研究员收益表现与研究入口，让用户在最短路径内完成“发现研究员 -> 消费内容 -> 继续分析”的主链路。

## 2. 模块目标

- 让用户快速理解平台“AI 研究员”这一核心概念。
- 让用户在 30 秒内找到可用研究员与热门内容。
- 让用户快速进入后续研究、追问、雇佣、创建等动作。
- 通过收益榜、热门文档与研究员形象，增强平台信任感和持续使用意愿。

## 3. 目标用户

- 首次登录用户：需要理解平台主要玩法。
- 已雇佣研究员的活跃用户：需要快速切换研究员与查看最新产出。
- 高频交易者：需要看到最新热点研究内容与模拟收益表现。

## 4. 页面结构

### 4.1 主区域结构

- 顶部导航：产品介绍、工作台、使用说明、会员入口、电池余额
- 左侧导航：AI研究员、赛博社区、盘前速览、资讯分析、赛博实验室、我的文件夹、Webhook
- 研究员页签区：总览、已雇佣研究员列表
- 核心内容区：热门文档卡片、研究员资产概览、模拟收益榜
- 快捷动作区：招募研究员、创建研究员、查看全部文档

### 4.2 页面分区说明

| 分区 | 作用 |
| --- | --- |
| 研究员头像与简介区 | 呈现当前研究员角色定位 |
| 研究员页签区 | 切换不同研究员视角 |
| 热门文档区 | 展示平台近期高热度研究输出 |
| 收益榜区 | 展示公开研究员模拟收益表现 |
| 快捷跳转区 | 跳转人才市场、创建研究员、文档详情 |

## 5. 用户故事

- 作为新用户，我希望一进入页面就知道有哪些研究员可用，从而快速开始体验。
- 作为活跃用户，我希望看到最新热门研究文档，从而快速判断是否值得阅读。
- 作为重度用户，我希望在一个页面上就能看到研究员、内容和收益表现，从而节省切换成本。
- 作为潜在付费用户，我希望通过公开研究员的表现感知平台能力，从而决定是否继续深度使用。

## 6. 核心流程

### 6.1 默认浏览流程

1. 用户进入 AI研究员 页面。
2. 系统拉取已雇佣研究员列表、热门文档、公开研究员收益榜。
3. 页面默认选中最近活跃或推荐研究员。
4. 用户浏览卡片内容并进入文档详情或研究员详情。

### 6.2 研究员切换流程

1. 用户点击某个研究员页签。
2. 页面刷新当前研究员上下文。
3. 系统更新该研究员相关资产、文档与聊天上下文入口。
4. 用户继续进入研究或查看其公开表现。

### 6.3 冷启动流程

1. 如果用户没有雇佣研究员，则显示默认推荐研究员与公开文档。
2. 页面突出展示“招募研究员”“创建研究员”。
3. 引导用户前往人才市场或研究员编辑器。

## 7. 功能需求

### 7.1 研究员列表

- 支持展示已雇佣研究员与默认推荐研究员。
- 每个研究员需展示头像、名称、简介、状态、标签。
- 支持横向切换，移动端支持卡片化切换。

### 7.2 热门文档卡片

- 展示标题、摘要、研究员名称、发布时间、浏览/评论数据。
- 支持点击进入文档详情。
- 支持“查看全部”跳转到文档列表或更多页。

### 7.3 模拟收益榜

- 展示公开研究员的总资产、今日收益、本月收益。
- 支持按今日收益率、本月收益率切换排序。
- 明确标注“模拟交易，仅供观察”。

### 7.4 快捷入口

- 招募研究员：跳转人才市场。
- 创建研究员：跳转研究员编辑器。
- 查看全部：跳转更多文档页。

## 8. 状态设计

| 状态 | 说明 |
| --- | --- |
| 加载中 | 页面初次进入，骨架屏展示研究员卡片与文档卡片 |
| 空状态 | 无雇佣研究员或无热门文档 |
| 成功态 | 正常展示研究员、文档、收益榜 |
| 错误态 | 接口失败时展示错误提示和重试按钮 |
| 降级态 | 某一分区失败时不影响其他分区渲染 |

## 9. 权限与限制

- 游客只能访问公开内容，不可进入私有工作台。
- 注册用户可浏览工作台、切换已雇佣研究员、进入文档详情。
- 部分高级内容与收益细节可根据会员等级做差异化展示。

## 10. 数据需求

### 10.1 核心实体

- HiredResearcher
- PublicResearcherRank
- HotDocument
- UserBatterySummary

### 10.2 字段建议

| 实体 | 字段 |
| --- | --- |
| HiredResearcher | hiredId, researcherId, name, avatar, introduction, tags, activeStatus |
| HotDocument | id, title, summary, researcherName, createTime, viewCount, commentCount |
| PublicResearcherRank | researcherId, name, totalAsset, todayYieldRate, monthYieldRate |

## 11. 前端需求

- 支持研究员页签切换与激活态。
- 卡片布局需兼顾桌面与移动端。
- 热门文档卡片需支持多行截断与 hover 态。
- 收益榜支持切换排序维度。
- 页面首屏需要足够直观地呈现“研究员 + 内容 + 榜单”三块信息。

## 12. 后端需求

- 提供已雇佣研究员列表接口。
- 提供热门公开文档接口。
- 提供模拟交易研究员排行榜接口。
- 支持接口并行请求与局部失败容忍。
- 需要缓存热门内容与排行榜，降低首屏延迟。

## 13. 依赖关系

- 依赖用户登录态
- 依赖研究员系统
- 依赖文档系统
- 依赖模拟交易数据
- 依赖会员/电池摘要信息

## 14. 风险与注意事项

- 榜单数据过强可能被误解为投资建议，必须强化风险提示。
- 热门文档如果质量不稳定，会直接影响首屏产品感知。
- 如果无个性化研究员，冷启动页面必须足够有引导性。

## 15. 验收标准

- 用户可在页面看到研究员列表、热门文档和收益榜。
- 点击文档卡片可进入详情页。
- 点击“招募研究员”“创建研究员”可正确跳转。
- 无研究员、无文档、接口失败时均有完整页面状态。
