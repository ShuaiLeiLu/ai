# 模块 PRD：账户中心、会员体系与电池系统

## 1. 模块定位

账户中心、会员体系与电池系统是整个平台的用户管理与商业化支撑模块，负责承接身份、账户安全、资源配额、付费能力、电池流水和邀请激励等功能。

## 2. 模块目标

- 提供清晰的用户账户与安全管理入口。
- 通过会员权益与电池机制形成稳定商业模式。
- 让用户明确知道自己拥有多少资源、消耗了什么、升级能得到什么。

## 3. 子模块范围

- 个人中心
- 安全设置
- 电池明细
- 会员套餐页
- 邀请好友页

## 4. 用户故事

- 作为用户，我希望看到我的昵称、手机、会员状态、电池余额等信息。
- 作为付费转化用户，我希望清楚不同套餐的差异与价值。
- 作为重度用户，我希望看到电池消耗明细，避免资源不透明。
- 作为增长用户，我希望通过邀请获得奖励。

## 5. 功能需求

### 5.1 个人中心

- 查看昵称、头像、手机号、注册时间、最后登录时间、账户状态。
- 支持编辑资料。
- 支持退出登录。

### 5.2 安全设置

- 修改密码
- 更换绑定手机
- 预留更多安全能力，例如设备管理、登录记录

### 5.3 电池明细

- 展示当前余额、永久电池、时效电池
- 展示月度用量
- 展示流水明细：时间、描述、变化值、余额

### 5.4 会员套餐

- 展示注册版、基础版、大师版、旗舰版
- 展示月付、年付切换
- 展示权益差异、赠送电池、功能解锁项
- 支持立即购买入口

### 5.5 邀请好友

- 展示邀请链接
- 支持复制邀请链接
- 展示邀请人数、开通 VIP 人数、注册奖励进度
- 展示邀请奖励规则与邀请记录

## 6. 数据模型

| 实体 | 字段 |
| --- | --- |
| User | id, nickname, avatar, phone, status, createTime, lastLoginTime |
| MembershipPlan | id, name, price, cycle, batteryGift, features |
| MembershipOrder | id, userId, planId, amount, status, payTime |
| BatteryAccount | userId, permanentBattery, temporaryBattery, totalBalance |
| BatteryLog | id, userId, amount, balance, changeType, description, createTime |
| InvitationRecord | id, inviterId, inviteeId, rewardType, rewardAmount, status |

## 7. 前端需求

- 个人中心页签清晰：个人信息、电池明细、安全设置。
- 套餐页要重点突出套餐差异与推荐方案。
- 电池明细列表要可分页并区分正负变化。
- 邀请页要突出奖励规则和复制动作。

## 8. 后端需求

- 用户信息查询与更新接口
- 安全设置相关接口
- 电池账户与流水接口
- 会员套餐与订单接口
- 邀请链接生成、邀请记录查询与奖励发放接口

## 9. 风险与注意事项

- 账户安全是底线能力，密码与手机更换要有二次校验。
- 电池流水必须可审计。
- 套餐页文案与功能限制必须与实际一致，避免承诺不一致。

## 10. 验收标准

- 用户可查看并编辑个人信息。
- 用户可进入安全设置修改密码或更换手机。
- 用户可查看电池余额与明细。
- 用户可查看套餐权益和购买入口。
- 用户可复制邀请链接并查看邀请记录。
