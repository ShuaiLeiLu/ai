# 模块 PRD：任务编排与自驱任务

## 1. 模块定位

任务编排模块负责将 AI 研究员从“被动问答”升级为“主动执行”的自动化系统。它包括系统级任务编排能力和研究员级自驱任务能力，是平台长期留存、自动化使用深度和电池消耗增长的重要支点。

## 2. 模块目标

- 帮助用户把重复性研究任务自动化。
- 支持按时间、条件和流程触发 AI 研究工作。
- 让多个研究员能够协同产出研究结果。
- 让任务输出可沉淀为文档、日志或提醒消息。

## 3. 模块范围

- 研究员自驱任务配置
- 系统级任务列表展示
- 调度表达式配置
- 输出类型配置
- 交易日限制配置
- 任务结果留存与回溯

## 4. 典型场景

- 每日盘前市场速览
- 每日盘后复盘
- 公告发布后自动分析
- 题材异动后自动追踪
- 某股票相关新闻聚合后自动生成研究结论
- 条件满足时自动推送到 Webhook

## 5. 用户故事

- 作为活跃用户，我希望每天固定时间自动生成复盘或盘前总结。
- 作为主题研究用户，我希望某个题材异动时自动生成跟踪报告。
- 作为研究员作者，我希望为研究员配置“持续输出”的公共任务。
- 作为运营方，我希望任务结果可以回溯，便于问题定位和效果分析。

## 6. 调度模式

### 6.1 一次性定时

- 用户指定某一天某一时间执行一次。
- 适合事件型、临时型研究任务。

### 6.2 固定间隔

- 每隔 N 分钟执行一次。
- 适合高频监控任务。

### 6.3 Cron 表达式

- 支持复杂定时表达。
- 适合盘前、盘后、工作日固定任务。

### 6.4 交易日限制

- 开启后仅在交易日执行。
- 可依赖交易日历服务。

## 7. 功能需求

### 7.1 任务配置

- 任务标题
- 任务类型
- 执行时间 / 间隔 / Cron
- 是否仅交易日
- 是否强制输出文档
- 任务描述
- 动态变量支持

### 7.2 动态变量

支持在任务标题或任务提示词中使用：

- `{{date}}`
- `{{lastDate}}`
- `{{nextDate}}`
- `{{lastTradeDate}}`
- `{{nextTradeDate}}`

### 7.3 任务执行结果

- 支持输出文档
- 支持输出工作日志
- 支持输出普通结果文本
- 支持与 Webhook 联动推送

### 7.4 任务列表

- 展示任务名称、触发方式、创建时间、最近执行结果、历史记录
- 支持查看执行历史
- 支持启停、编辑、删除

## 8. 执行流程

1. 用户创建任务。
2. 系统校验调度配置是否合法。
3. 系统入库并交给调度器。
4. 到达触发条件时，任务被派发给指定研究员或工作流。
5. 执行完成后记录状态、日志、结果文档和耗电情况。
6. 如绑定 Webhook，则同步推送结果。

## 9. 状态设计

| 状态 | 含义 |
| --- | --- |
| DRAFT | 草稿态，未启用 |
| ACTIVE | 已启用，待触发 |
| RUNNING | 正在执行 |
| SUCCESS | 执行成功 |
| FAILED | 执行失败 |
| PAUSED | 暂停执行 |
| DELETED | 逻辑删除 |

## 10. 数据模型

| 实体 | 字段 |
| --- | --- |
| Task | id, ownerId, researcherId, title, scheduleType, scheduleConfig, enabled |
| TaskExecution | id, taskId, triggerTime, startTime, endTime, status, resultType |
| TaskExecutionLog | id, executionId, level, content, createTime |
| TaskOutput | id, executionId, outputType, documentId, payload |

## 11. 前端需求

- 配置界面要降低 Cron 复杂度，提供示例与说明。
- 任务列表需要清晰显示“时间触发”“事件触发”等类型。
- 执行历史支持翻页或时间线视图。
- 失败时展示失败原因和重试入口。
- 任务编辑需支持沿用原配置。

## 12. 后端需求

- 提供任务创建、更新、删除、启停、列表、执行历史接口。
- 调度器需支持不同触发类型。
- 任务执行需具备超时控制、幂等与重试机制。
- 输出文档与日志需与文档系统、日志系统打通。
- 交易日规则需接入交易日历。

## 13. 风险与注意事项

- 高频任务可能造成电池消耗过快，需要预算和限流机制。
- Cron 配置复杂，需避免用户配置错误导致失控触发。
- 长任务失败或重复执行会影响用户信任，必须有清晰状态和日志。

## 14. 验收标准

- 用户能创建 3 类调度任务。
- 用户能查看任务列表与执行历史。
- 任务执行结果能落为日志或文档。
- 任务可启停、编辑、删除。
- 失败任务有明确错误提示和日志。
