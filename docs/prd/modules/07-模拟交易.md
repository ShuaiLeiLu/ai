# 模块 PRD：模拟交易

## 1. 模块定位

模拟交易模块用于将 AI 研究员的研究结论映射到模拟持仓、交易日志和历史调仓表现之中，帮助用户观察研究策略的执行效果，但不提供真实交易功能。

## 2. 模块目标

- 展示 AI 研究员策略执行过程。
- 帮助用户理解研究结论是否具备操作一致性。
- 形成“研究 -> 执行 -> 复盘”的闭环。
- 以强风险提示方式明确其仅为观察与学习用途。

## 3. 页面结构

- 交易组切换
- 账户总资产/可用资金/收益摘要
- 当前持仓
- 调仓记录
- 实时交易日志
- 历史数据
- 重要风险提示弹窗

## 4. 用户故事

- 作为用户，我希望看到 AI 研究员如何实际执行策略，而不是只看文字结论。
- 作为研究者，我希望观察同一研究员的调仓逻辑和持仓变化。
- 作为平台方，我希望通过模拟交易增强产品可看性，但避免被误解为荐股。

## 5. 功能需求

### 5.1 账户摘要

- 总资产
- 今日盈亏
- 收益率
- 持仓资金
- 可用资金

### 5.2 当前持仓

- 股票名称、代码、数量、成本、现价、盈亏、收益率
- 标记当日已卖出状态

### 5.3 调仓记录

- 建仓时间
- 平仓时间
- 收益率
- 盈亏金额
- 当前是否已卖出

### 5.4 交易日志

- 支持实时 SSE 流式推送
- 支持只看 `TRADE`
- 支持清空日志显示
- 支持按时间分组展示

### 5.5 历史数据

- 支持按交易组查看历史持仓快照或历史策略表现

## 6. 风险提示需求

- 首次进入页面必须展示强提醒弹窗。
- 明确说明模拟盘仅用于观察 AI 对金融场景的学习与理解，不构成投资建议。
- 页面显著位置持续展示风险提示。

## 7. 数据模型

| 实体 | 字段 |
| --- | --- |
| TradeGroup | id, name, researcherId, status |
| MockAccount | tradeGroupId, totalAssets, availableCash, todayProfit, profitRate |
| MockPosition | id, tradeGroupId, stockCode, stockName, quantity, avgPrice, currentPrice, profitLoss |
| MockTradeRecord | id, tradeGroupId, stockCode, openDate, closeDate, returnRate, profitLoss |
| MockTradeLog | id, tradeGroupId, level, message, timestamp |

## 8. 前端需求

- 实时日志区需要支持时间线样式和自动滚动。
- 当前持仓与历史调仓切换要清晰。
- 移动端需要通过弹层或全屏模式查看日志与历史数据。
- 风险提示弹窗需只在首次进入时强提示，但页面内仍保留持续提示。

## 9. 后端需求

- 提供交易组列表接口。
- 提供账户摘要接口。
- 提供当前持仓接口。
- 提供历史调仓记录接口。
- 提供交易日志 SSE 接口。
- 提供健康检查与状态接口。

## 10. 风险与注意事项

- 该模块最容易引发误导，所有收益数据必须标注为模拟。
- 需要避免使用与真实券商过于类似的术语和视觉误导。
- 实时日志大量推送时需控制前端性能。

## 11. 验收标准

- 用户可以切换交易组并看到对应账户摘要。
- 用户可以查看当前持仓和历史调仓。
- 日志可以实时接收并过滤 `TRADE`。
- 首次进入有明显风险提示。
