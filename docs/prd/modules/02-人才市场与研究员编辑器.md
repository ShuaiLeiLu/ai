# 模块 PRD：人才市场与研究员编辑器

## 1. 模块定位

该模块负责承接平台的“研究员供给侧”与“用户个性化配置侧”。人才市场解决研究员发现与雇佣问题，研究员编辑器解决研究员创建、配置、测试、发布与版本化管理问题，两者共同构成赛博投研的角色化 AI 核心能力。

## 2. 模块目标

- 帮助用户快速发现、理解、雇佣适合自己的研究员。
- 让用户能低门槛创建自己的 AI 研究员。
- 让研究员具备可配置的技能、知识库、MCP、自驱任务和测试能力。
- 建立研究员模板、版本、发布与市场供给的闭环。

## 3. 子模块范围

- 人才市场
- 我的研究员
- 研究员编辑器
- 研究员发布/下架/版本
- 研究员模板查看

## 4. 人才市场页面结构

- 搜索栏
- 推荐与公开研究员列表
- 卡片信息：头像、名称、简介、等级、雇佣量、版本、标签
- 卡片动作：雇佣、解雇、查看模板、查看简历
- 无限滚动加载或分页加载

## 5. 编辑器页面结构

- 提示词设定区
- 基本信息区：名称、头像、介绍
- 技能选择区
- 知识库选择区
- MCP 服务器选择区
- 自驱任务配置区
- 对话测试区
- 保存 / 发布操作区

## 6. 用户故事

- 作为普通用户，我希望在人才市场中按风格找到合适的研究员并一键雇佣。
- 作为进阶用户，我希望创建自己的研究员，让它按照我的投资方法工作。
- 作为研究员作者，我希望保存草稿、反复测试、逐步完善后再发布到市场。
- 作为平台运营，我希望对研究员供给进行管理，确保市场质量稳定。

## 7. 核心流程

### 7.1 雇佣流程

1. 用户进入人才市场。
2. 浏览或搜索研究员。
3. 点击卡片查看简历或模板。
4. 点击雇佣。
5. 系统校验登录状态、会员限制、研究员额度。
6. 雇佣成功后刷新工作台研究员列表。

### 7.2 解雇流程

1. 用户在人才市场或已雇佣状态中点击解雇。
2. 系统弹出二次确认。
3. 明确提示聊天记录与使用链路可能受影响。
4. 用户确认后执行解雇。

### 7.3 创建研究员流程

1. 用户进入创建研究员页面。
2. 填写名称、介绍、提示词。
3. 选择头像、技能、知识库、MCP。
4. 添加自驱任务。
5. 进行对话测试。
6. 保存草稿。
7. 发布到个人可用或公开市场。

### 7.4 测试流程

1. 用户在编辑器中输入测试问题。
2. 系统通过测试流式接口返回研究员回复。
3. 用户观察思考过程、回复风格和工具调用逻辑。
4. 用户继续优化提示词与配置。

## 8. 功能需求

### 8.1 人才市场

- 支持搜索研究员名称与描述。
- 支持模板公开标识、等级标识、版本标识、雇佣量展示。
- 支持雇佣、解雇、查看模板、查看简历。
- 支持分页或无限滚动加载。
- 支持区分“平台研究员”“用户自建研究员”。

### 8.2 我的研究员

- 展示当前用户创建的研究员列表。
- 空状态下提供“创建研究员”引导。
- 已创建研究员支持编辑、发布、下架、复制。

### 8.3 编辑器基础能力

- 支持提示词编辑，长度上限可配置。
- 支持名称、头像、介绍必填校验。
- 支持保存草稿。
- 支持未保存状态提醒。

### 8.4 技能/知识库/MCP 挂载

- 技能最多选择 25 个。
- 可绑定多个知识库。
- 可绑定多个 MCP 服务。
- 挂载状态需要在保存或发布时清晰反馈进度。

### 8.5 自驱任务

- 任务标题可选。
- 支持一次性定时、固定间隔、Cron。
- 支持仅交易日执行。
- 支持强制输出文档。
- 支持任务描述与动态变量。
- 最多可配置 10 个自驱任务。

### 8.6 发布与版本

- 保存不等于发布。
- 发布后才能对已雇佣研究员生效或上架市场。
- 支持版本号与发布时间管理。
- 支持下架与取消公开。

## 9. 状态与校验

| 场景 | 状态 |
| --- | --- |
| 编辑器初次进入 | 加载中 |
| 新建无内容 | 空白编辑态 |
| 表单未通过 | 字段校验错误态 |
| 保存中 | 进度弹窗或保存中态 |
| 发布中 | 发布处理中态 |
| 测试对话中 | 流式回复态 |
| 雇佣/解雇 | 操作中禁用按钮 |

## 10. 数据模型

| 实体 | 字段 |
| --- | --- |
| Researcher | id, authorId, name, avatar, introduction, prompt, visibility, marketVersion, templateVisible |
| ResearcherSkillRelation | researcherId, skillId |
| ResearcherKnowledgeBaseRelation | researcherId, knowledgeBaseId |
| ResearcherMcpRelation | researcherId, mcpServerId |
| ResearcherTask | id, researcherId, title, scheduleType, date, interval, cron, isTradingDayOnly, forceDocument, prompt |
| ResearcherPublishRecord | id, researcherId, version, publishTime, status |

## 11. 前端需求

- 编辑器应分区明确，避免表单过长造成认知负担。
- 自驱任务编辑采用弹窗或抽屉形式，支持编辑与删除。
- 测试对话区支持流式输出、思考过程和错误提示。
- 技能/知识库/MCP 选择器支持搜索、已选状态和数量限制。
- 保存与发布要明确区分。

## 12. 后端需求

- 提供研究员列表、详情、创建、更新、删除、复制、发布、下架接口。
- 提供雇佣、解雇接口。
- 提供技能、知识库、MCP 可选项查询接口。
- 提供研究员测试对话流式接口。
- 提供创建/更新过程中挂载进度回传机制。

## 13. 风险与注意事项

- 提示词配置复杂，若缺少模板与示例会提高创建门槛。
- 雇佣/解雇涉及聊天记录与上下文影响，需要有清晰提示。
- 发布机制必须避免草稿误发布。
- 多种外挂能力叠加后，研究员行为可控性与稳定性需要重点评估。

## 14. 验收标准

- 用户能在人才市场完成搜索、雇佣、解雇。
- 用户能创建研究员并保存草稿。
- 用户能为研究员绑定技能、知识库、MCP。
- 用户能添加自驱任务并通过校验。
- 用户能进行测试对话并看到流式返回。
- 用户能完成发布并看到版本变化。
