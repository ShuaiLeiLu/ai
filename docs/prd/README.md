## 使用方式

- 正式技术架构选型文档用于统一技术栈、分层规范、基础设施与实施标准。
- 产品体验审计用于区分真功能与占位/演示能力，辅助产品规划与研发优先级判断。
- 总 PRD 用于产品立项、设计评审、研发排期与项目管理。
- 模块文档用于单模块设计、研发拆分、测试验收与 AI 协作开发。
- 每个模块文档末尾均附带前端与后端 AI 提示词，可直接用于生成代码任务或研发子需求。

## 推荐阅读顺序

1. 先阅读总 PRD，理解产品目标、用户、模块边界与业务闭环。
2. 阅读正式技术架构选型文档，统一前后端技术栈与工程标准。
3. 阅读产品体验审计，明确当前真功能与占位/演示边界。
4. 再按研发优先级阅读模块文档，建议顺序为：AI研究员主链路 -> 盘前速览/资讯分析 -> 文档/社区 -> 自动化/模拟交易 -> 生态与商业化。
5. 在进入开发阶段时，直接复制对应模块文档中的 AI 提示词给前端或后端 AI 执行。
