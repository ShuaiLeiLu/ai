# 后端工厂模式架构说明

## 目标

通过统一的模块协议和注册器，把“模块发现、路由聚合、启动钩子收集”放到同一处管理，减少在主路由文件中手工维护 `include_router` 的重复操作。

## 模块协议

定义位置：`server/app/modules/base_module.py`

- `AppModule`（协议）
  - `name: str`：模块唯一名称，用于注册与查询。
  - `router: APIRouter`：模块对外暴露的路由。
  - `startup: StartupHook | None`：可选启动钩子。
- `ModuleDefinition`：协议的默认实现（推荐用于模块封装）。

## 核心组件

- `ModuleRegistry`
  - `register()` / `register_many()`：注册模块，按 `name` 去重。
  - `get()` / `has()` / `list()`：查询模块。
- `ModuleFactory`
  - `build_router()`：聚合注册器中的模块路由。
  - `startup_hooks()`：收集所有非空 `startup` 钩子。
- `create_default_registry()`
  - 默认注册内置模块（当前包含 `system` 与 `auth`）。

## 示例模块封装

- `server/app/modules/system/module.py`
  - 暴露 `module = ModuleDefinition(name="system", router=router)`
- `server/app/modules/auth/module.py`
  - 暴露 `module = ModuleDefinition(name="auth", router=router, startup=_startup)`

## 如何新增模块并接入工厂

1. 在新模块目录创建 `module.py`，封装模块对象：

```python
from app.modules.base_module import ModuleDefinition
from app.modules.example.router import router


async def _startup() -> None:
    ...


module = ModuleDefinition(name="example", router=router, startup=_startup)
```

2. 在 `create_default_registry()` 中注册新模块：

```python
from app.modules.example.module import module as example_module

registry.register_many([system_module, auth_module, example_module])
```

3. 在主路由中接入工厂聚合（建议替换手工 include）：

```python
from app.modules.base_module import build_api_router

api_router = build_api_router()
```

4. 如需执行启动逻辑，在应用 `lifespan` 中运行钩子：

```python
from app.modules.base_module import collect_startup_hooks

for hook in collect_startup_hooks():
    result = hook()
    if inspect.isawaitable(result):
        await result
```

## 基础验证

测试文件：`server/tests/test_factory_bootstrap.py`

- 校验注册器可获取 `system` 模块。
- 校验工厂聚合后可访问 `/api/v1/health`。
- 校验仅收集到声明了 `startup` 的模块钩子（当前为 `auth`）。
