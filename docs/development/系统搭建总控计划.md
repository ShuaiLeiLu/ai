## 4. 后端执行顺序（主线）
### 阶段 1（已完成）
- 所有模块路由、Schema、Service 完成可联调版本（当前为内存数据源）。

### 阶段 2（下一步）
- 持久化改造：PostgreSQL + SQLAlchemy 模型落地
- 关键表：用户、研究员、文档、任务、资讯、社区帖子、笔记、Webhook、会员与电池流水、模拟交易账户
- Alembic 迁移脚本完善

### 阶段 3
- 异步任务体系：Celery worker + beat
- 事件流：任务执行日志、研究员生成流程、资讯入库 pipeline
- SSE 实时推送联调

### 阶段 4
- 对外依赖接入：行情/资讯源、LLM 供应商、对象存储
- 容灾与限流、审计日志、灰度策略

## 5. 联调规则
- 所有接口统一返回结构：
  - `{"success": true, "data": ...}`
- 列表统一：
  - `{"items": [...], "total": n}`
- 错误语义：
  - 400 参数错误
  - 401 未认证
  - 404 资源不存在
  - 409 资源冲突

## 6. 外部环境清单（需要你准备）
- 必选：
  - Docker Desktop（用于 PostgreSQL/Redis/MinIO）
  - Node.js 20+
  - Python 3.11+
  - pnpm 9+
- 生产级建议：
  - 资讯源 API Key（新闻与公告）
  - 行情源 API Key（实时/历史行情）
  - LLM API Key（OpenAI 或兼容网关）
  - 短信/邮件服务（登录验证码与通知）
